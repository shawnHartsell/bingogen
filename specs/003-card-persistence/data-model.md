# Data Model: Card Persistence & Navigation

**Feature Branch**: `003-card-persistence`  
**Date**: 2026-03-21  
**Storage**: MongoDB (local instance for development) — native `mongodb` driver

## Entities

### BingoCard (MongoDB Document)

The primary persisted entity. Stored as a single JSON document in the `cards` collection. The full document is replaced on every save (whole-document replacement strategy).

**Collection**: `cards`

| Field             | Type     | Description                                        | Constraints                                                                          |
| ----------------- | -------- | -------------------------------------------------- | ------------------------------------------------------------------------------------ |
| `_id`             | ObjectId | Unique card identifier (auto-generated by MongoDB) | Primary key. Used in URL as `/cards?id=<_id>`. Contains embedded creation timestamp. |
| `title`           | string   | User-provided card title                           | Required. Non-empty. Displayed in sidebar and card header. Editable after creation.  |
| `cells`           | Cell[]   | Ordered array of 25 cells (flat, row-major)        | Exactly 25 elements. Index 12 is always the free space.                              |
| `completedBingos` | number[] | Indices of completed bingo lines (0–11)            | Derived from cell completion state. Persisted to avoid recalculation.                |
| `createdAt`       | Date     | Timestamp of card creation                         | Auto-set on insert. Used for sidebar sort order (newest first).                      |
| `updatedAt`       | Date     | Timestamp of last modification                     | Auto-set on every save.                                                              |

### Cell (Embedded Sub-document)

Embedded within the `cells` array of a `BingoCard` document. Not a separate collection.

| Field         | Type    | Description                           | Constraints                                  |
| ------------- | ------- | ------------------------------------- | -------------------------------------------- |
| `row`         | number  | Row position (0–4)                    | 0 ≤ row ≤ 4                                  |
| `col`         | number  | Column position (0–4)                 | 0 ≤ col ≤ 4                                  |
| `goalTitle`   | string  | Display text for the cell's goal      | Max 100 characters. "FREE" for free space.   |
| `isFreeSpace` | boolean | Whether this is the center free space | true only for index 12                       |
| `isCompleted` | boolean | Whether the cell is checked off       | Always true for free space                   |
| `notes`       | string  | Markdown content for cell notes       | Max 500 characters. Empty string by default. |

### CardListItem (Derived / Read Projection)

Not a separate collection. A projection of `BingoCard` used to populate the sidebar list efficiently, avoiding loading full cell arrays for every card.

| Field       | Type     | Description            |
| ----------- | -------- | ---------------------- |
| `_id`       | ObjectId | Card identifier        |
| `title`     | string   | Card title for display |
| `createdAt` | Date     | For sort ordering      |

**MongoDB projection**: `{ _id: 1, title: 1, createdAt: 1 }`

## Relationships

```
CardList (sidebar)
  └── 1:N → BingoCard (each item links to a full card)
                └── 1:25 → Cell (embedded array, always exactly 25)
```

- The sidebar loads `CardListItem` projections (lightweight).
- Selecting a card loads the full `BingoCard` document by `_id`.
- Cells are always embedded — never stored or queried independently.

## Validation Rules

| Rule                                      | Source                       | Enforcement                              |
| ----------------------------------------- | ---------------------------- | ---------------------------------------- |
| Title must be non-empty                   | FR-003                       | Application-level validation before save |
| Title must be a string                    | FR-002                       | TypeScript type + runtime check          |
| Cells array must have exactly 25 elements | Existing domain logic        | Application-level validation             |
| Cell `goalTitle` max 100 chars            | `MAX_GOAL_LENGTH` constant   | Application-level validation             |
| Cell `notes` max 500 chars                | `MAX_NOTES_LENGTH` constant  | Application-level validation             |
| Cell index 12 must be free space          | `FREE_SPACE_INDEX` constant  | Application-level validation             |
| `completedBingos` values must be 0–11     | `TOTAL_BINGO_LINES` constant | Derived from `detectBingos()`            |

## State Transitions

```
[Goal Entry] --( GENERATE_CARD + title )--> [New Card]
    |                                            |
    |                                       ( save to DB )
    |                                            |
    |                                            v
    |                                     [Saved Card]
    |                                       /   |   \
    |                          (toggle)   (notes) (title edit)
    |                              \       |       /
    |                               v     v     v
    |                             [Updated Card]
    |                                    |
    |                              ( save to DB )
    |                                    |
    |                                    v
    |                             [Saved Card] (loop)
    |
    |                             [Saved Card]
    |                                    |
    |                              ( delete )
    |                                    |
    |                                    v
    |                              [Removed]
```

## Mapping to Existing Types

The existing `AppState` interface will be extended to support persistence. The `Cell` interface remains unchanged.

### New Types (additions to `lib/types.ts`)

```typescript
// Persisted card document (what gets stored in MongoDB)
export interface BingoCardDocument {
  _id?: string; // MongoDB ObjectId as string
  title: string; // User-provided card title
  cells: Cell[]; // 25 cells, same as existing Cell interface
  completedBingos: number[]; // Completed bingo line indices
  createdAt: Date;
  updatedAt: Date;
}

// Lightweight projection for sidebar list
export interface CardListItem {
  _id: string;
  title: string;
  createdAt: Date;
}

// Extended AppState for persistence awareness
export interface AppState {
  // Existing fields...
  goals: string[];
  cells: Cell[];
  cardGenerated: boolean;
  completedBingos: number[];
  newBingos: number[];

  // New fields for persistence
  cardId: string | null; // MongoDB _id of the active card (null before first save)
  cardTitle: string; // Editable card title
}
```

### New Actions (additions to `AppAction` union)

```typescript
export type AppAction =
  // Existing actions...
  | { type: "ADD_GOAL"; goal: string }
  | { type: "REMOVE_GOAL"; index: number }
  | { type: "GENERATE_CARD" }
  | { type: "TOGGLE_COMPLETION"; cellIndex: number }
  | { type: "UPDATE_NOTES"; cellIndex: number; notes: string }
  | { type: "UPDATE_GOAL_TITLE"; cellIndex: number; title: string }
  | { type: "RESET" }
  // New actions for persistence
  | { type: "SET_CARD_TITLE"; title: string }
  | { type: "LOAD_CARD"; card: BingoCardDocument };
```
